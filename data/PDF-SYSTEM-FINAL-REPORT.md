## 🏗️ 系统架构

### 数据流架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户入口                              │
│                  products.html                           │
└────────────────────┬────────────────────────────────────┘
                     │ 点击产品链接
                     ▼
      ┌──────────────────────────────┐
      │   产品详情页面 (9 个)         │
      │  power_inductors.html 等     │
      │  [17-19 KB, HTML5 + CSS]    │
      └────────────┬─────────────────┘
                   │ 加载产品数据
                   ▼
      ┌──────────────────────────────┐
      │   数据库层                    │
      │  products-pdf-map.json       │
      │  [13.5 KB, 126 个 PDF]      │
      └────────────┬─────────────────┘
                   │ 获取 PDF 列表
                   ▼
      ┌──────────────────────────────┐
      │   PDF 存储                    │
      │  Active/ 文件夹               │
      │  [共 126 个 PDF 文件]        │
      └──────────────────────────────┘
```


### 核心特性

#### 1. 多语言支持

## 🚀 部署和使用

### 快速开始

1. **启动本地服务器**
   ```bash
   cd /Volumes/Extreme\ Pro/Datatronic/06-Website/website
   python3 -m http.server 8000
   ```

2. **访问产品页面**
   ```
   http://localhost:8000/power_inductors.html
   http://localhost:8000/current_sense.html
   ... (其他产品)
   ```

3. **从 products.html 访问**
   - 更新 products.html 中的链接指向本地页面
   - 用户即可点击产品链接访问详情页面

### 维护指南

#### 添加新的 PDF

1. 编辑 `data/products-pdf-map.json`
2. 在对应产品的 `pdfs` 数组中添加：
   ```json
   {
     "series": "4260 Series",
     "filename": "4260_series.pdf"
   }
   ```
3. 将 PDF 文件复制到 `Active/` 文件夹
4. 可选：重新生成页面
   ```bash
   python3 generate_product_pages.py
   ```

#### 修改产品信息

1. 编辑 `data/products-pdf-map.json`
2. 修改 `en_name`、`jp_name`、`cn_name` 或 `description_*` 字段
3. 重新生成页面（推荐）

#### 修改页面样式

1. 编辑各产品页面的 `<style>` 部分
2. 或编辑 `product-detail-template.html` 后重新生成所有页面

## 📞 技术支持

### 常见问题

**Q: 如何添加新产品类别？**
```
A: 编辑 products-pdf-map.json，添加新产品对象到 products 数组，
   然后运行 generate_product_pages.py 脚本自动生成页面。
```

**Q: PDF 无法下载怎么办？**
```
A: 检查：
   1. PDF 文件是否存在于 Active/ 文件夹
   2. 文件名是否与 JSON 中的名称完全匹配
   3. 文件是否有读取权限
```

**Q: 如何修改页面的颜色？**
```
A: 编辑 HTML 文件中的 <style> 部分，修改 CSS 颜色值。
   主要颜色定义在 rgb(216, 26, 41) 处。
```

**Q: 如何支持新语言？**
```
A: 1. 在 products-pdf-map.json 中为新语言添加字段
   2. 在 i18next 配置中添加新语言的翻译
   3. 在语言按钮中添加新语言
   4. 重新生成页面
```

### 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| PDF 无法下载 | 文件不存在或权限问题 | 检查 Active/ 文件夹和文件权限 |
| 页面显示为空 | JavaScript 错误 | 检查浏览器控制台的错误信息 |
| 语言显示乱码 | 字符编码问题 | 确保 HTML 文件的 UTF-8 编码 |
| 链接不工作 | 文件路径错误 | 验证 href 属性中的路径 |
